# Краткое резюме проекта

## Что это

Это реальный кейс развёртывания сайта на **Flute CMS** на VPS.

Проект оформлен не как абстрактная теория, а как понятный deployment summary:

- сайт размещён на Ubuntu VPS
- фронт отдаёт Nginx
- PHP работает через PHP-FPM
- данные хранятся в MariaDB
- используется SSL
- фоновые задачи запускаются через CRON

## Цель репозитория

Показать:

1. как выглядит нормальная установка Flute CMS на VPS;
2. какие настройки критичны для корректной работы;
3. как оформить это в GitHub так, чтобы репозиторий выглядел как полноценный технический кейс.

## Основная мысль

Самая важная техническая деталь при развёртывании Flute CMS — это правильный **document root**.

Нужно, чтобы веб-сервер смотрел не в корень проекта, а в папку:

```bash
public/
```

Именно это чаще всего ломает установку, маршруты и часть страниц при первом деплое.

## Использованный подход

В этом репозитории описан deployment под классический VPS-стек:

- Ubuntu
- Nginx
- PHP-FPM
- MariaDB
- SSL
- CRON

Это хороший формат как для рабочего сайта, так и для портфолио DevOps / Linux / Web Infra.
